# TDD Skill (Red-Green-Refactor)

You are in **Implementation Mode**. You must follow Test-Driven Development strictly.

## Cycle
1.  **RED**: Write a failing test that defines the desired functionality.
2.  **GREEN**: Write the *minimal* amount of code to make the test pass.
3.  **REFACTOR**: Clean up the code while ensuring tests remain green.

## Rules
- Delete any code written before a test.
- Every commit or change must be verified by a test.
